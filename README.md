# Hana Poster AI | 花屋さんのポスター文字入れ

花屋さん向けに、カテゴリ棚型ポスターギャラリーから用途を選び、文字入れ・PNG/PDF保存・仮注文ができる試作Webアプリです。花の情報確認・AI販促文作成・注文管理も搭載しています。

---

## 主な機能

### 花図鑑
- 花名・読み方・英名で検索
- 花詳細表示（育て方・花言葉など）
- 複数写真の切り替え
- 出典・ライセンス表示

### AI販促文作成（v0.6〜 本物のAI API対応）
- OpenAI API による店頭POP文・SNS文・補足文の生成
- `OPENAI_API_KEY` が未設定の場合はローカル固定ロジックにフォールバック
- フォールバック時は画面に通知を表示
- 生成中はボタンを無効化し「AIが文章を作成中です...」を表示

### ポスターギャラリー（v1.0-rc3〜）
- 春・夏・秋・冬・母の日・ギフト・開店祝いの7カテゴリを常時表示
- 各カテゴリに縮小ポスターサムネイルを横一列で表示
- 検索入力不要 — 見ながらクリックするだけ
- サムネイルクリックで即ポスター編集へ反映（テキスト・スタイル自動入力）
- 選択中サムネイルに「選択中」バッジ表示
- 「もっと見る」でそのカテゴリ行だけ追加表示（6枚ずつ）
- テンプレートは `static/poster_templates.json` で管理（差し替え可能）

### ポスター文字入れ
- 正式素材（ユーザーアップロード画像）への文字入れ
- 素材の権利チェック（外部写真はPNG保存・PDF保存・発注不可）
- 仕上がり確認プレビュー
- PNG保存
- PDF保存（jsPDF によるブラウザ側生成）
- 仮注文確認

### 文字スタイル6種類（v1.0-rc5〜）
- **文字なし** — 画像のみ、文字を一切表示しない
- **小さく入れる** ― 写真の隅に小さく控えめに（初期推奨）
- **上品文字** ― 半透明カード・細フォントで花屋向けの上品な印象
- **強めPOP** ― 太文字・高コントラストでフェア名をしっかり見せる
- **全面告知** ― セール・母の日・イベント向けの大きな訴求文字
- **下余白** ― 写真の下余白に文字を入れ、画像本体を壊さない

### 文字サイズ調整スライダー（v1.0-rc5〜）
- タイトル・サブタイトル・店舗名日付の文字サイズを 70〜130% で個別調整
- AI文字配置提案クリック時も倍率を自動設定
- 仕上がり確認・PNG保存・PDF保存・注文JSONにも反映

### 編集画面 sticky プレビュー（v1.0-rc5〜）
- PC表示でスライダー操作中もポスタープレビューが常時表示
- 詳細設定をスクロールしながらリアルタイムで仕上がりを確認可能

### PDF出力（v0.9〜）
- 編集画面・仕上がり確認画面・仮注文完了画面からPDF保存
- 管理画面の保存済み注文からもPDF保存
- 正式素材使用時のみPDF保存可能（外部写真・参考素材は不可）
- A4縦は portrait、A4横は landscape でPDFページサイズを自動設定
- その他サイズ（正方形・16:9・9:16・元画像比率）はA4内に収まる比率維持
- 注文JSONに `pdf_filename` フィールドを記録
- PDF末尾に簡易版注意文を自動付与

### AI修正リクエスト（v0.6〜 本物のAI API対応）
- 現在のタイトル・サブタイトル・補足文 + 修正コメントをもとにAIが修正案を生成
- API未接続時はローカル固定ロジックにフォールバック
- 修正案ラベルに「AI修正案」「修正案（ローカル生成）」を区別表示

### 仮注文・ファイル紐付け（v0.4〜）
- 仮注文時に `order_id`（YYYYMMDD_HHMMSS形式）を自動生成
- 注文内容をJSONファイルとしてダウンロード可能
- PNGを同じ `order_id` 付きファイル名で保存可能
- PDFを同じ `order_id` 付きファイル名で保存可能（v0.9〜）
- JSON内に `png_filename` / `pdf_filename` を記録し、後からJSON・PNG・PDFを対応付け可能

### 仮注文履歴一覧（v0.5〜）
- 仮注文するたびに履歴をブラウザの localStorage に自動保存
- 履歴を新しい順に一覧表示
- 各履歴からJSONを再保存可能
- 個別履歴の削除・全履歴の一括削除
- ページ再読み込み後も履歴を保持

### サーバー側注文保存（v0.7〜）
- 仮注文完了後に「サーバーに保存」ボタンでサーバーへ保存
- 注文JSONとポスターPNGを `orders/` フォルダに保存
- `orders/` フォルダは起動時に自動作成・`.gitignore` 除外済み
- 「一覧を更新」ボタンで最新状態を取得

### サーバー保存済み注文の管理（v0.8〜）
- サーバー保存済み注文の一覧表示（注文ID・日時・花名・店舗名・タイトル・サイズ・使い方・ファイル名・ステータス）
- 「詳細を見る」ボタンでモーダル表示
  - ポスターPNGプレビュー
  - 注文JSONの主要項目
  - チェック項目の確認状況
  - 素材状態
- ステータス管理（select で変更 → サーバーに即保存）
- カラーバッジによるステータス視認
- 注文削除（JSONファイル・PNGファイルを同時削除）

## デモ版クイックスタート

**AIキーなしでもすべての基本機能が使えます。**

### 手順

**1. リポジトリをクローン**

```bash
git clone https://github.com/Hajime-na/ai-hana-zukan.git
cd ai-hana-zukan
```

**2. 依存パッケージをインストール**

```bash
pip install -r requirements.txt
```

**3. サーバーを起動**

```bash
uvicorn main:app --reload
```

ブラウザで開く：`http://127.0.0.1:8000`

---

### 使い方の流れ

```
① 花を選ぶ
   左側の花図鑑から花名をクリック → 詳細・花言葉・写真を確認

② 販促文を作る（任意）
   「この花で販促文を作る」→ 用途・雰囲気を選んで「AI案を作る」
   → 気に入った文を「ポスターに反映」でポスターへ転送

③ ポスターを作る
   「この花でポスターを作る」→ 正式素材をアップロード
   → タイトル・店舗名・日付を入力 → プレビューを確認

④ 確認・保存
   「仕上がり確認へ」→ 内容をチェック
   → PNGで保存 / PDFで保存 / この内容で仮注文する
```

> ポスター編集・PNG/PDF保存・仮注文は**正式素材（自分でアップロードした画像）**でのみ使用できます。  
> 花図鑑の参考写真では保存・発注できません。

---

### AIなしで使う場合

`.env` ファイルを作成しなければ、AI機能はローカル固定ロジックにフォールバックします。  
画面に「AI APIに接続できなかったため、ローカル生成文を表示しています。」と表示されますが、  
ポスター作成・PNG保存・PDF保存・仮注文は**すべて正常に動作します。**

## AI API 設定

### 必要なパッケージ

```bash
pip install -r requirements.txt
```

### 環境変数の設定

`.env.example` をコピーして `.env` を作成し、APIキーを設定してください。

```bash
cp .env.example .env
```

`.env` の内容：

```
OPENAI_API_KEY=your_api_key_here
OPENAI_MODEL=gpt-4.1-mini
```

> `.env` は `.gitignore` により Git 管理対象外です。APIキーを GitHub に push しないでください。

### APIキー未設定時の動作

`OPENAI_API_KEY` が未設定またはAPIが失敗した場合、アプリは起動し続け、ローカル固定ロジックで販促文を生成します。

> AI APIに接続できなかったため、ローカル生成文を表示しています。

## API エンドポイント一覧

### `POST /api/generate-copy`

| フィールド | 内容 |
|------------|------|
| `flower_name` | 花名 |
| `flower_features` | 花の特徴 |
| `flower_language` | 花言葉 |
| `care_summary` | 育て方メモ |
| `purpose` | 用途（店頭POP / Instagram など） |
| `tone` | 雰囲気（やさしい / 高級感 など） |
| `current_main_title` | 現在のメインタイトル（修正時） |
| `current_subtitle` | 現在のサブタイトル（修正時） |
| `current_note` | 現在の補足文（修正時） |
| `user_request` | 修正コメント（空の場合は新規生成） |

出力：`{main_title, subtitle, note, instagram_text, pop_text}`

`user_request` が空 → 新規生成 / 非空 → 修正モード として自動判定します。

---

### `POST /api/save-order`

| フィールド | 内容 |
|------------|------|
| `order_id` | 注文ID（YYYYMMDD_HHMMSS） |
| `order_data` | 注文データ（JSONオブジェクト） |
| `png_data_url` | ポスターPNG（`data:image/png;base64,...` 形式） |

成功レスポンス：`{ "ok": true, "order_id": "...", "json_file": "...", "png_file": "..." }`

失敗レスポンス：`{ "ok": false, "error": "..." }`

---

### `GET /api/orders`

`orders/` フォルダ内の `order_*.json` を新しい順に読み込み一覧を返します。

返却フィールド：`order_id` / `created_at` / `flower_name` / `shop_name` / `main_title` / `poster_size` / `image_usage` / `status` / `json_file` / `png_file`

---

### `GET /api/orders/{order_id}/poster`

`orders/poster_{order_id}.png` を画像として返します。

---

### `GET /api/orders/{order_id}/json`

`orders/order_{order_id}.json` を返します。

---

### `POST /api/orders/{order_id}/status`

注文JSON内の `status` フィールドを更新します。

入力：`{ "status": "checking" }` など

成功レスポンス：`{ "ok": true, "order_id": "...", "status": "..." }`

---

### `DELETE /api/orders/{order_id}`

`order_{order_id}.json` と `poster_{order_id}.png` を削除します。

成功レスポンス：`{ "ok": true, "order_id": "..." }`

## PDF出力仕様

### 生成方式

- ブラウザ側で [jsPDF](https://github.com/parallax/jsPDF) (v2.5.1) を使用
- 既存のポスタープレビューをCanvas経由で画像化し、PDFへ貼り込み
- サーバーにPDFは保存しない（ブラウザダウンロードのみ）

### PDFページサイズ

| ポスターサイズ | PDFページサイズ | 向き |
|---|---|---|
| A4縦 | 210 × 297 mm | portrait |
| A4横 | 297 × 210 mm | landscape |
| 正方形・16:9・9:16・元画像比率 | A4内に収まる比率維持のカスタムサイズ | 比率により自動判定 |

### 注意事項

- **PDF出力は簡易版**です
- 印刷会社への正式入稿前には、解像度・塗り足し・色味・用紙サイズを必ず確認してください
- PDF/X / 塗り足し / トンボ / CMYK変換 には**未対応**です
- ブラウザ表示と印刷物の色味は異なる場合があります

## ポスター画像の管理

### 保存場所

```
static/posters/
  ├── spring_001.png
  ├── spring_002.png
  ├── summer_001.png
  ├── mothersday_001.png
  └── ...
```

- **フォルダ分けはしない** — `static/posters/` に全ポスター画像をフラットに置く
- カテゴリはファイル名プレフィックスと `poster_templates.json` の `categories` フィールドで管理する

### ファイル名規則

| プレフィックス | カテゴリ |
|---|---|
| `spring_` | 春 |
| `summer_` | 夏 |
| `autumn_` | 秋 |
| `winter_` | 冬 |
| `mothersday_` | 母の日 |
| `gift_` | ギフト |
| `opening_` | 開店祝い |

連番は3桁ゼロ埋め（`001`, `002`, ...）

### poster_templates.json への登録例

```json
{
  "id": "spring-a",
  "short_label": "春A",
  "image": "/static/posters/spring_001.png",
  "categories": ["春"],
  "season": "春",
  "poster_allowed": true
}
```

- `image`: `/static/posters/xxx.png` 形式で指定
- `categories`: 複数カテゴリに同時所属可能（例：`["春", "ギフト", "かわいい"]`）
- `season`: 季節の主カテゴリ（表示ソート用）
- `poster_allowed: true` — 正式販売用
- `poster_allowed: false` — デモ・検討中

### カテゴリ運用の考え方

1枚の画像を複数カテゴリに出せるよう、フォルダではなく `categories` 配列で分類する。

```json
"categories": ["春", "ギフト", "かわいい"]
```

この場合、春・ギフト・かわいいの3カテゴリ棚に同じ画像が表示される。

- 本番運用ではサーバー保存・注文管理画面が必要です
