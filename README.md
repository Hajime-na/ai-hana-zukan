# AI花図鑑｜花屋さんの販促・ポスター作成ツール

花屋さん向けに、花の情報確認・販促文作成・ポスター文字入れ・正式素材アップロード・PNG保存・仮注文確認・仮注文履歴一覧・サーバー側注文保存までできる試作Webアプリです。

---

## 主な機能

### 花図鑑
- 花名・読み方・英名で検索
- 花詳細表示（育て方・花言葉など）
- 複数写真の切り替え
- 出典・ライセンス表示

### AI販促文作成（v0.6〜 本物のAI API対応）
- OpenAI API による店頭POP文・SNS文・補足文の生成
- `OPENAI_API_KEY` が未設定の場合はローカル固定ロジックにフォールバック
- フォールバック時は画面に通知を表示
- 生成中はボタンを無効化し「AIが文章を作成中です...」を表示

### ポスター文字入れ
- 正式素材（ユーザーアップロード画像）への文字入れ
- 素材の権利チェック（外部写真はPNG保存・発注不可）
- 仕上がり確認プレビュー
- PNG保存
- 仮注文確認

### AI修正リクエスト（v0.6〜 本物のAI API対応）
- 現在のタイトル・サブタイトル・補足文 + 修正コメントをもとにAIが修正案を生成
- API未接続時はローカル固定ロジックにフォールバック
- 修正案ラベルに「AI修正案」「修正案（ローカル生成）」を区別表示

### 仮注文・ファイル紐付け（v0.4〜）
- 仮注文時に `order_id`（YYYYMMDD_HHMMSS形式）を自動生成
- 注文内容をJSONファイルとしてダウンロード可能
- PNGを同じ `order_id` 付きファイル名で保存可能
- JSON内に `png_filename` を記録し、後からJSONとPNGを対応付け可能

### 仮注文履歴一覧（v0.5〜）
- 仮注文するたびに履歴をブラウザの localStorage に自動保存
- 履歴を新しい順に一覧表示
- 各履歴からJSONを再保存可能
- 個別履歴の削除・全履歴の一括削除
- ページ再読み込み後も履歴を保持

### サーバー側注文保存（v0.7〜）
- 仮注文完了後に「サーバーに保存」ボタンでサーバーへ保存
- 注文JSONとポスターPNGを `orders/` フォルダに保存
- `orders/` フォルダは起動時に自動作成・`.gitignore` 除外済み
- 「サーバー保存済み注文」セクションで一覧表示
- 「一覧を更新」ボタンで最新状態を取得

## AI API 設定

### 必要なパッケージ

```bash
pip install -r requirements.txt
```

### 環境変数の設定

`.env.example` をコピーして `.env` を作成し、APIキーを設定してください。

```bash
cp .env.example .env
```

`.env` の内容：

```
OPENAI_API_KEY=your_api_key_here
OPENAI_MODEL=gpt-4.1-mini
```

> `.env` は `.gitignore` により Git 管理対象外です。APIキーを GitHub に push しないでください。

### APIキー未設定時の動作

`OPENAI_API_KEY` が未設定またはAPIが失敗した場合、アプリは起動し続け、ローカル固定ロジックで販促文を生成します。

> AI APIに接続できなかったため、ローカル生成文を表示しています。

## API エンドポイント一覧

### `POST /api/generate-copy`

| フィールド | 内容 |
|------------|------|
| `flower_name` | 花名 |
| `flower_features` | 花の特徴 |
| `flower_language` | 花言葉 |
| `care_summary` | 育て方メモ |
| `purpose` | 用途（店頭POP / Instagram など） |
| `tone` | 雰囲気（やさしい / 高級感 など） |
| `current_main_title` | 現在のメインタイトル（修正時） |
| `current_subtitle` | 現在のサブタイトル（修正時） |
| `current_note` | 現在の補足文（修正時） |
| `user_request` | 修正コメント（空の場合は新規生成） |

出力：`{main_title, subtitle, note, instagram_text, pop_text}`

`user_request` が空 → 新規生成 / 非空 → 修正モード として自動判定します。

---

### `POST /api/save-order`

| フィールド | 内容 |
|------------|------|
| `order_id` | 注文ID（YYYYMMDD_HHMMSS） |
| `order_data` | 注文データ（JSONオブジェクト） |
| `png_data_url` | ポスターPNG（`data:image/png;base64,...` 形式） |

成功レスポンス：

```json
{ "ok": true, "order_id": "...", "json_file": "order_....json", "png_file": "poster_....png" }
```

失敗レスポンス：

```json
{ "ok": false, "error": "..." }
```

---

### `GET /api/orders`

`orders/` フォルダ内の `order_*.json` を新しい順に読み込み一覧を返します。

返却フィールド：`order_id` / `created_at` / `flower_name` / `shop_name` / `main_title` / `poster_size` / `image_usage` / `json_file` / `png_file`

## 画像素材の扱い

| 種別 | 用途 | PNG保存 | 仮注文 |
|------|------|---------|--------|
| 外部写真（Unsplash等） | 花図鑑の参考表示のみ | 不可 | 不可 |
| ユーザーアップロード素材 | ポスター文字入れ | 可 | 可 |

> 本番公開前に、写真内容・ライセンス表記が正しいか必ず確認してください。

---

## 画像の使い方（2モード）

- **写真に文字を入れる** ― アプリ側で文字帯・タイトル・店舗名・日付を重ねます
- **画像をそのまま使う** ― 写真をそのままポスター素材として使用します
