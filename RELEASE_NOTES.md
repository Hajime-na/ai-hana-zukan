## v1.1-rc3 — 見積もり・請求メモ管理

**タグ**: `v1.1-rc3`  
**ベース**: v1.1-rc2  
**検証**: PASS

### このリリースについて

注文ごとの見積もり情報を `estimate` フィールドとして注文 JSON に独立保存できるようにしたリリース。管理者メモ（`admin_note`）も別フィールドで保持し、お客様向け案内文には含めない設計。`estimate` と `payment` は独立した API で保存され、互いに上書きしない。

### 追加機能

#### 注文JSONに `estimate` フィールドを追加

`POST /api/orders/{id}/estimate` で以下を保存できます：

| フィールド | 内容 |
|---|---|
| `design_fee` | 制作費 |
| `print_fee` | 印刷費 |
| `shipping_fee` | 送料 |
| `tax` | 消費税 |
| `discount` | 値引き |
| `total` | 合計（自動計算・手動上書き可） |
| `estimate_note` | 見積もりメモ（`estimate` 内に保存） |

`admin_note`（管理者メモ）は `estimate` とは別にルートフィールドとして保存。

#### `POST /api/orders/{order_id}/estimate`

新規エンドポイント。成功レスポンス：`{ "ok": true, "order_id": "...", "estimate": { ... } }`

`estimate` と `payment` は独立して保存されるため、一方を更新しても他方は上書きされない。

#### 管理画面の注文詳細に「見積もり・請求メモ」セクション追加

- 制作費・印刷費・送料・消費税・値引きを入力
- 合計は入力値から自動計算（制作費＋印刷費＋送料＋消費税－値引き）
- 見積もりメモ（テキストエリア）
- 管理者メモ（テキストエリア、お客様向け案内文に含まれない）
- 「見積もりを保存」ボタンでサーバーへ即保存

#### 見積もり合計を支払い合計に反映

「見積もり合計を支払い合計に反映」ボタンで、見積もりの合計金額を支払い情報セクションの合計金額フィールドへワンクリック転記。

#### お客様向け案内文コピー

「お客様向け案内文をコピー」ボタンで以下を含む案内文をクリップボードへコピー：

- 注文内容（ポスター・印刷サイズ・用紙・配送方法・印刷確認ID）
- お見積り（制作費・印刷費・送料・消費税・合計）
  - 値引きがある場合のみ値引き行を追加
- お支払い（支払い方法・支払い状態・支払期限・決済リンク）

`admin_note` は案内文に含まれない。

#### `GET /api/orders` に見積もり情報を追加

`estimate_total` を一覧レスポンスに追加。

#### 注文一覧カードに見積もり合計バッジを表示

`estimate_total` が設定された注文に「見積：¥○○○○」バッジを表示。`amount_total`（支払い合計）と異なる値の場合は両方を並べて表示。

## v1.1-rc2 — 支払い方法・入金ステータス管理

**タグ**: `v1.1-rc2`  
**ベース**: v1.1-rc1  
**検証**: 12/12 PASS

### このリリースについて

決済API連携ではなく、手動運用向けの支払い方法・入金ステータス管理を実装したリリース。注文ごとの支払い情報を注文JSONに保存し、管理画面から操作できる。

### 追加機能

#### 注文JSONに `payment` フィールドを追加

仮注文保存後、`POST /api/orders/{id}/payment` で以下を保存できます：

| フィールド | 内容 |
|---|---|
| `method` | 支払い方法 |
| `status` | 支払いステータス |
| `amount_total` | 合計金額 |
| `amount_design` | デザイン費 |
| `amount_print` | 印刷費 |
| `amount_shipping` | 送料 |
| `amount_tax` | 消費税 |
| `payment_due_date` | 支払期限 |
| `paid_at` | 入金日 |
| `payment_note` | 支払いメモ |
| `payment_link` | 決済リンク URL |

#### 支払い方法（6種類）

未選択 / 銀行振込 / PayPay / Square決済リンク / Stripe決済リンク / その他

#### 支払いステータス（6種類）

| ステータス | バッジ色 |
|---|---|
| 未請求 | グレー |
| 請求済み | 青 |
| 入金待ち | 黄 |
| 入金済み | 緑 |
| 返金 | オレンジ |
| キャンセル | 赤 |

#### 管理画面の注文詳細に「支払い情報」セクション追加

- 支払い方法・ステータスを select で変更
- 金額内訳（合計・デザイン費・印刷費・送料・消費税）を入力
- 支払期限・入金日（日付ピッカー）
- 決済リンク（URL入力）
- 支払いメモ（テキストエリア）
- 「支払い情報を保存」ボタンでサーバーへ即保存

#### `GET /api/orders` に支払い情報を追加

`payment_status` / `payment_method` / `amount_total` を一覧レスポンスに追加。

#### 注文一覧カードに支払い状態と合計金額を表示

- 支払いステータスバッジ（6色対応）
- 合計金額（¥表示、未設定時は非表示）

#### Prio発注用メモに管理用支払い情報を追加

```
---管理用---
支払い状態：入金済み
支払い方法：銀行振込
合計金額：15000
```

モーダル上で最新入力値を読み取ってコピーするため、保存前の状態もメモに反映される。

#### Slack通知（支払いステータス変更時）

ステータスが変更された場合のみ通知。通知内容：注文ID・印刷確認ID・店舗名・支払い方法・変更前→後・合計金額。

Slack未設定（`SLACK_NOTIFY_ENABLED=false`）・通知失敗時も保存処理は継続します。

#### `POST /api/orders/{order_id}/payment`

新規エンドポイント。成功レスポンス：`{ "ok": true, "order_id": "...", "payment": { ... } }`


Slack 通知に失敗しても注文保存・ステータス変更は失敗しません。サーバーログに失敗内容を記録して処理を継続します（`urllib.request` 使用、追加パッケージ不要）。

### 追加・変更内容

#### poster_master.csv をマスターにした同期運用

- `poster_master.csv` がテンプレートデータの唯一の正（単一ソース・オブ・トゥルース）
- `scripts/sync_poster_templates.py` が `poster_master.csv` を読み込み、SAMPLE 入りプレビューを生成しつつ `static/poster_templates.json` を再生成する
- `start_server.bat` がサーバー起動前に自動で `sync_poster_templates.py` を実行するため、CSV を更新するだけでWeb表示に即反映される
- `/api/poster-templates` は `Cache-Control: no-store` で最新 JSON を毎回配信する

#### 同期フロー

```
poster_master.csv
  ↓ scripts/sync_poster_templates.py（起動時自動実行）
static/poster_templates.json + static/posters_preview/*.jpg
  ↓ /api/poster-templates（no-store）
Web画面のカテゴリ棚
```

#### poster_master.csv のフィールド

| フィールド | 内容 |
|---|---|
| `poster_id` | HP-XXXX 形式の一意ID |
| `source_path` | 高解像度元画のパス |
| `title` / `short_label` | 表示名 |
| `season` / `categories` / `tags` | 分類 |
| `poster_allowed` / `featured` | 公開制御 |
| `default_main_title` / `default_subtitle` / `default_note` | デフォルト文字 |
| `poster_type` / `poster_design` / `poster_position` | スタイル設定 |
| `flower_match` | 対応する花名 |

## v1.0-rc5 — 編集UX改善版

**タグ**: `v1.0-rc5`  
**ベース**: v1.0-rc4 (安定地点)  
**コミット**: `7d51c9b`

### このリリースについて

ポスター編集画面のUXを中心に改善したリリースです。文字スタイルを6種類に再設計し、文字サイズの個別調整スライダーを追加しました。またPC表示での編集作業性を大きく改善し、詳細設定をスクロールしながら常にプレビューを確認できるようになりました。

### 追加・変更内容

#### 文字スタイル6種類

| スタイル | 特徴 |
|---|---|
| 文字なし | 画像のみ。文字を一切表示しない |
| 小さく入れる | 写真の隅に控えめに入れる（初期推奨） |
| 上品文字 | 半透明カード・細フォントで上品な印象 |
| 強めPOP | 太文字・高コントラストでフェア名を訴求 |
| 全面告知 | セール・イベント向けの大きな白文字 |
| 下余白 | 写真の下余白に入れ、画像本体を壊さない |

旧スタイル名（`direct` / `pop-band`）は後方互換マッピングで新名称に自動変換。

#### AI文字配置提案の更新

提案ラベルと設定値を新スタイルに対応：
- **写真を活かす案** ― minimal / タイトル 90%
- **上品文字案** ― elegant / タイトル 100%
- **強めPOP案** ― strong-pop / タイトル 115%

#### 文字サイズ調整スライダー

詳細設定内に3つのスライダーを追加（初期値100%、範囲70〜130%）：
- タイトル文字サイズ
- サブタイトル文字サイズ
- 店舗名・日付文字サイズ

スライダーの値はDOMプレビュー・キャンバス出力・仕上がり確認・注文JSONすべてに反映。リセットボタンでまとめて100%に戻せる。

#### 仕上がり確認画面の整理

- 信頼性のない拡大確認エリア（画質確認・文字品質確認）を非表示化
- 確認チェック項目を右カラムから左カラム（素材警告の直下）へ移動し、視認しやすく整理
- 編集画面の保存ボタンを非表示のまま維持（保存は仕上がり確認画面に集約）

#### 編集画面 sticky プレビュー（PC）

- PC（901px以上）でポスタープレビューが画面上部に固定
- 詳細設定・文字サイズスライダー・テキストオフセットを操作中もプレビューを常時確認可能
- JavaScriptによる scroll 追従実装（`overflow-x: clip` 対応）
- モバイルでは通常レイアウト（固定なし）

### 既存機能への影響

変更なし。以下はすべて正常動作を確認：

- 棚サムネイルクリック・テンプレート選択
- AI販促文・AI修正リクエスト
- PNG保存 / PDF保存 / 仮注文 / サーバー保存
- 注文管理UI（一覧・詳細・ステータス・削除）
- SAMPLE透かし・保存不可制御（poster_allowed: false）

- PNG保存 / PDF保存 / 仮注文 / サーバー保存 / 注文管理UI

- AI販促文・AI修正リクエスト
